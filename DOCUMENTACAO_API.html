🔐 AUTENTICAÇÃO

POST /auth/register
Registrar novo usuário
Cria uma nova conta de usuário
Request Body
{ "name": "João Silva", "email": "joao@example.com", "password": "senha123", "phone": "(11) 98765-4321", "cpf": "123.456.789-00", "address": "Rua das Flores, 123" }
Response (201 Created)
{ "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": 1, "name": "João Silva", "email": "joao@example.com", "role": "USER" } }
Possíveis Erros
  • 400: Dados inválidos ou usuário já existe
POST /auth/login
Fazer login
Autentica um usuário e retorna token JWT
Request Body
{ "email": "joao@example.com", "password": "senha123" }
Response (200 OK)
{ "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": 1, "name": "João Silva", "email": "joao@example.com", "role": "USER" } }
GET /auth/me 🔒 AUTH
Obter perfil do usuário
Retorna informações do usuário autenticado
Headers
Authorization: Bearer {token}
Response (200 OK)
{ "id": 1, "name": "João Silva", "email": "joao@example.com", "phone": "(11) 98765-4321", "cpf": "123.456.789-00", "address": "Rua das Flores, 123", "role": "USER" }
PUT /auth/me 🔒 AUTH
Atualizar perfil
Modifica informações do perfil autenticado

{ "name": "João Silva Atualizado", "phone": "(11) 99999-9999", "address": "Rua Nova, 456" }
Response (200 OK)
{ "id": 1, "name": "João Silva Atualizado", "email": "joao@example.com", "phone": "(11) 99999-9999", "role": "USER" }

🐾 ANIMAIS

GET /animals
Listar animais
Lista todos os animais com filtros opcionais
Query Parameters (Opcionais)
Parâmetro Descrição Exemplo
species Espécie do animal cachorro, gato
size Tamanho pequeno, medio, grande
status Status disponivel, adotado, apadrinhado
q Busca textual labrador, rex
Exemplos de Uso
/animals?species=cachorro&status=disponivel /animals?q=labrador /animals?size=grande
Response (200 OK)
[ { "id": 1, "name": "Rex", "species": "cachorro", "breed": "Labrador", "gender": "macho", "ageYears": 3, "size": "grande", "description": "Cão amigável e dócil", "temperament": "Calmo", "vaccinated": true, "neutered": true, "status": "disponivel", "photoUrl": "https://example.com/rex.jpg", "shelterId": 1, "createdAt": "2024-01-15T10:30:00Z" } ]
GET /animals/{id}
Obter animal por ID
Retorna detalhes de um animal específico
Path Parameters
Parâmetro Tipo Descrição
id integer ID do animal
POST /animals 👨‍💼 ADMIN
Criar novo animal
Cria um novo registro de animal (requer ADMIN)
Headers
Authorization: Bearer {token_admin} Content-Type: application/json
Request Body
{ "name": "Luna", "species": "gato", "breed": "Siamês", "gender": "fêmea", "ageYears": 2, "size": "pequeno", "description": "Gata meiga e carinhosa", "temperament": "Brincalhona", "vaccinated": true, "neutered": true, "status": "disponivel", "photoUrl": "https://example.com/luna.jpg", "shelterId": 1 }
Response (201 Created)
{ "id": 2, "name": "Luna", "species": "gato", ... }
PUT /animals/{id} 👨‍💼 ADMIN
Atualizar animal completamente
Atualiza todos os campos de um animal (requer ADMIN)
ℹ️ Nota: Use PATCH para atualizar apenas alguns campos
PATCH /animals/{id} 👨‍💼 ADMIN
Atualizar animal parcialmente
Atualiza apenas os campos informados (requer ADMIN)
Request Body (exemplo)
{ "status": "adotado", "description": "Recém adotado - muito feliz!" }
DELETE /animals/{id} 👨‍💼 ADMIN
Deletar animal
Remove um animal do sistema (requer ADMIN)
Response (204 No Content)

Sem corpo de resposta

📝 ADOÇÕES

POST /adoptions 🔒 AUTH
Solicitar adoção
Usuário autenticado solicita a adoção de um animal
Request Body
{ "animalId": 1, "motivation": "Procuro um cão amigável para minha família", "experience": "Tenho experiência com cães", "homeType": "casa" }
Response (201 Created)
{ "id": 1, "animal": { "id": 1, "name": "Rex" }, "adopter": { "id": 1, "name": "João Silva" }, "status": "PENDENTE", "createdAt": "2024-01-20T15:30:00Z" }
GET /adoptions/mine 🔒 AUTH
Minhas adoções
Retorna todas as adoções do usuário autenticado
GET /adoptions 👨‍💼 ADMIN
Listar todas as adoções
Lista todas as solicitações de adoção (requer ADMIN)
Query Parameters
Parâmetro Valores
status PENDENTE, APROVADO, RECUSADO, CONCLUIDO
PATCH /adoptions/{id}/review 👨‍💼 ADMIN
Revisar adoção
Admin aprova, recusa ou conclui uma adoção
Request Body
{ "status": "APROVADO" }
Status Aceitos
  • APROVADO: Aprova a adoção
  • RECUSADO: Recusa a adoção
  • CONCLUIDO: Marca como concluída

👶 APADRINHAMIENTOS

POST /godparents 🔒 AUTH
Apadrinhar um animal
Cria um novo apadrinhamamiento para um usuário
Request Body
{ "animalId": 1 }
Response (201 Created)
{ "id": 1, "animal": { "id": 1, "name": "Rex" }, "status": "ATIVO", "startedAt": "2024-01-20T16:00:00Z" }
GET /godparents/mine 🔒 AUTH
Listar meus apadrinhamientos
Retorna todos os apadrinhamientos do usuário autenticado
PATCH /godparents/{id}/status 🔒 AUTH
Desativar apadrinhamamiento
Altera o status do apadrinhamamiento para INATIVO
ℹ️ Nota: Apenas o proprietário pode desativar seu apadrinhamamiento
DELETE /godparents/{id} 🔒 AUTH
Deletar apadrinhamamiento
Remove um apadrinhamamiento (apenas o proprietário)
Response (204 No Content)

Sem corpo de resposta

ℹ️ Nota: Apenas o proprietário do apadrinhamamiento pode deletá-lo

🔐 Notas de Segurança

  • Todos os endpoints protegidos requerem um token JWT válido no header Authorization: Bearer {token}
  • Endpoints marcados com 👨‍💼 ADMIN requerem role ADMIN
  • Usuários regulares só podem acessar seus próprios dados
  • Use Content-Type: application/json para POST, PUT, PATCH
